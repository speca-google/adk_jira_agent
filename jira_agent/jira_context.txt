## IMPORTANT JQL NOTES
*   **Dynamic User Queries:** Use `currentUser()` to find issues related to you. For example, `assignee = currentUser()` finds all issues assigned to you.
*   **Finding Empty Fields:** Use `IS EMPTY` to find issues where a field has no value. For example, `assignee IS EMPTY` finds unassigned issues. Use `IS NOT EMPTY` to find issues where a field has any value.
*   **Exact Match (`=`) vs. Text Contains (`~`):** Use `=` for exact matches on fields like `status` or `priority` (e.g., `priority = High`). Use `~` for partial text searches within text fields like `summary` or `description` (e.g., `summary ~ "database"`).
*   **Searching for Multiple Values:** Use the `IN` operator to check a field against a list of values, which is cleaner than multiple `OR` clauses. For example, `status in ("To Do", "In Progress")`. Use `NOT IN` to exclude a list of values.
*   **Dynamic Date Functions:** Use functions like `now()`, `startOfDay()`, `startOfWeek()`, `startOfMonth()`, and `startOfYear()` to create queries relative to the current date. You can also use relative dates like `-1w` (one week ago). For example, `created >= startOfMonth()` finds all issues created since the beginning of the current month.

## EXAMPLES

**Question:** "Show me all high or highest priority work in any Migration project that is not yet done, and sort it by project and then priority."
**JQL Query:** "project in (JM, MM, TM, WM) AND priority in (High, Highest) AND statusCategory != Done ORDER BY project, priority DESC"

**Question:** "Find all Stories across the Launch projects that were created in the last 90 days but have not been estimated with Story Points."
**JQL Query:** "project in (LL, ML, WL) AND issuetype = Story AND \"Story Points\" is EMPTY AND created >= -90d"

**Question:** "What issues are assigned to me in the Platform or Integration projects that are either overdue or do not have a due date set?"
**JQL Query:** "project in (JP, MP, FI, RI, TI, WI) AND assignee = currentUser() AND (dueDate < now() OR dueDate is EMPTY)"

**Question:** "List all Epics that have been updated this month but have not yet been completed, ordered by the most recently updated."
**JQL Query:** "issuetype = Epic AND updated >= startOfMonth() AND status != Done ORDER BY updated DESC"

**Question:** "Find all tasks or subtasks in Project Zero still in a 'To Do' state that mention 'impact' or 'measure' in their summary."
**JQL Query:** "project = KAN AND issuetype in (Task, Subtask) AND statusCategory = \"To Do\" AND (summary ~ \"impact\" OR summary ~ \"measure\")"

**Question:** "Show me any issues that were marked as Resolved in the last 3 months but have since been Reopened, as this might indicate a quality issue."
**JQL Query:** "status = Reopened AND resolved >= -3M ORDER BY resolved DESC"

**Question:** "I need to find all high-priority 'orphan' Stories that are not linked to an Epic, so we can organize them in our backlog."
**JQL Query:** "issuetype = Story AND \"Epic Link\" is EMPTY AND priority in (High, Highest)"
