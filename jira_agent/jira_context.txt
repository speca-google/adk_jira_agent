## IMPORTANT JQL NOTES
* **Operators**:
    * Use `=` for exact matches (e.g., `status = 'To Do'`).
    * Use `~` for "CONTAINS" searches in text fields (e.g., `summary ~ "login"`).
    * Use `!=`, `!~`, `>`, `<`, `>`=, `<=` for comparisons.
* **Keywords**:
    * Use `IN` to match multiple values (e.g., `priority IN (High, Highest)`).
    * Use `IS EMPTY` or `IS NOT EMPTY` to find issues where a field is blank or not (e.g., `assignee IS EMPTY`).
* **Functions**:
    * `currentUser()`: Refers to the person running the query. (e.g., `reporter = currentUser()`).
    * `startOfDay()`, `startOfWeek()`, `startOfMonth()`, `startOfYear()`: Used for date comparisons. (e.g., `created >= startOfMonth()`). You can use offsets like `"-1w"` for "one week ago".
* **Logic**:
    * Combine clauses with `AND` and `OR`. Use parentheses `()` to group logic.
* **Sorting**:
    * Use `ORDER BY` to sort results (e.g., `ORDER BY priority DESC, created ASC`).
* **Custom Fields (Advanced)**:
    * **Required Syntax**: Custom fields **MUST** be referenced by their ID to ensure queries are reliable, as names can change. The required syntax is `cf[ID]`.
    * **Finding the Syntax**: The `Available Fields` section below provides the exact JQL to use for every searchable field. For custom fields, it will be in the `cf[12345]` format.
    * **Example**: To query the "Story point estimate" field, check the `Available Fields` list to find its JQL syntax (e.g., `cf[10016]`) and then use it: `cf[10016] >= 5`.
    * **Syntax Depends on Field Type**:
        * **Text**: Use the `~` operator. Example: `cf[10011] ~ "API"`.
        * **Number**: Use comparison operators (`=`, `>`, `<`). Example: `cf[10016] > 5`.
        * **Date/Time**: Use date functions. Example: `cf[10023] < "2025/01/01"`.
        * **Select Lists** (Single or Multi-select): Use `IN` or `NOT IN`. Example: `"Focus Areas" IN ("Mobile", "Backend")`.
        * **User Pickers**: Use `=` with the user's name or `currentUser()`. Example: `cf[10003] = currentUser()`.
        * **Dont need confirm with user the custom fields ID, only if you have two or more with same description.**

## EXAMPLES
**Question:** "Show me all unresolved Stories and Tasks with a 'High' or 'Highest' priority across all Migration projects (JM, MM, TM, WM) that currently have no assignee, ordered by priority."
**JQL Query:** "project in (JM, MM, TM, WM) AND issuetype in (Story, Task) AND status not in (Resolved, Closed, Done) AND priority in (High, Highest) AND assignee is EMPTY ORDER BY priority DESC, created ASC"

**Question:** "Find all issues in the Integration projects (FI, RI, TI, WI) or Launch projects (LL, ML, WL) that are currently flagged, are not yet Done, and have no value in the 'Team' custom field."
**JQL Query:** "project in (FI, RI, TI, WI, LL, ML, WL) AND cf[10021] is not EMPTY AND status != Done AND cf[10001] is EMPTY"

**Question:** "Which issues assigned to me are currently 'In Progress' but have a due date that has already passed, or have no due date at all?"
**JQL Query:** "assignee = currentUser() AND status = 'In Progress' AND (duedate < now() OR duedate is EMPTY)"

**Question:** "List all Stories in any 'Platform' or 'Launch' project that have a Story Point Estimate but are still in the 'To Do' status and were created more than 30 days ago."
**JQL Query:** "project in (JP, MP, LL, ML, WL) AND issuetype = Story AND cf[10016] is not EMPTY AND status = 'To Do' AND created < -30d ORDER BY cf[10016] DESC"

**Question:** "Find all Epics in the 'Montgomery Platform' (MP) or 'Wood-Conway Launch' (WL) projects that have a 'Target end' date set for this month."
**JQL Query:** "project in (MP, WL) AND issuetype = Epic AND cf[10023] >= startOfMonth() AND cf[10023] <= endOfMonth()"

**Question:** "Show me all issues that I have been watching which have been updated in the last 7 days, excluding any from the 'Project Zero' (KAN) project."
**JQL Query:** "watcher = currentUser() AND updated >= -7d AND project != KAN ORDER BY updated DESC"

**Question:** "Identify all tasks or subtasks that were resolved in the last two weeks but have comments indicating a problem (e.g., containing 'failed', 'issue', or 'problem')."
**JQL Query:** "issuetype in (Task, Subtask) AND resolutiondate >= -14d AND comment ~ 'failed OR issue OR problem' ORDER BY resolutiondate DESC"
